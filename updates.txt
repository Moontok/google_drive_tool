=== DESIGN / QUALITY ISSUES ===

6. sheets.py:345-411 and sheets.py:393-410 — local state mutates before the API call succeeds
   change_sheet_name_request and delete_sheet_request both update __current_sheets
   immediately, but the actual API change only happens when batch_update() is called.
   If batch_update() raises or is never called, local state diverges from Google.
   Consider mutating __current_sheets only inside batch_update() after a successful
   response, or document this clearly so callers know not to rely on the local state
   until after the batch.

8. formatting.py:9 — Color.PURPLE is actually magenta
   PURPLE = (1, 0, 1) is full red + full blue with no green, which is magenta.
   True purple is closer to (0.5, 0, 0.5).

9. formatting.py:82-148 — cell_font_format and chart_font_format are nearly identical
   The only difference is that cell_font_format has strikethrough and underline
   parameters. These could be one function: chart callers simply wouldn't pass those
   two arguments. Keeping two near-duplicate functions means any future change (e.g.,
   adding opacity) must be made in both places.

10. sheets.py throughout / formatting.py:63 — parameter named `range` shadows the built-in
    Many methods use `range` as a parameter name (process_range, fill_request,
    get_values, etc.), shadowing Python's built-in range(). Rename to range_str or
    cell_range throughout to avoid the shadow and improve clarity.

11. sheets.py:412 — hideGridlines parameter violates Python naming conventions
    The parameter should be hide_gridlines (snake_case) to match all other parameters
    in the file.

12. sheets.py:754-760 — _process_range_as_str has a wrong docstring
    The docstring says "Adds the request to requests pool. This will update when the
    next batch_update is called." — this is copied from batch methods and is completely
    wrong. The function is a pure utility that parses a string into a tuple.

13. chart.py:367 and chart.py:380 — LineChart methods hardcode "basicChart" instead of
    using self._chart_type
    set_series_line_style and set_line_smoothing access
    self._chart["spec"]["basicChart"] directly. While this isn't a functional bug
    today (LineChart always has _chart_type == "basicChart"), it's inconsistent with
    the rest of the chart methods, which use self._chart_type.
